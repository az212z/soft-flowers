## لماذا اخترنا هذا النشاط
محل بتقييم عالٍ (4.8 من 446) وصور منتجات حقيقية جذّابة (باقات، صناديق مخمل، واجهة مضيئة)، بلا موقع إلكتروني — فرصة قوية لتحويل الزوّار إلى طلبات عبر واتساب.

## فكرة التصميم — "ناعم وهوائي وباستيل"
لوحة هادئة: **خوخي ناعم + أخضر نعناعي + أبيض + رمادي دافئ**. خطوط عربية مزدوجة: **El Messiri** للعناوين و**Tajawal** للنص. حركة لطيفة (بتلات منجرفة في الهيرو، تفتّح الكروت عند التمرير، كين-بيرنز بطيء، تكبير عند المرور) تحترم `prefers-reduced-motion`. الهوية مميّزة ومختلفة عن بقية مواقع الدفعة.

## التقنية
HTML/CSS/JS فانيلا بدون أي build step أو إطار عمل أو `node_modules`. خطوط Google عبر preconnect + `display=swap`. أيقونات SVG inline.

## التشغيل محليًا
```bash
cd soft-flowers
python3 -m http.server 8080
# افتح http://localhost:8080
```

## حالة الـ Backend
**غير مربوط بخادم.** نموذج الطلب يبني رسالة واتساب جاهزة على `wa.me/966555398789`، ويحفظ نسخة من الطلب محليًا في `localStorage` (عرض توضيحي)، مع رسالة نجاح (toast). لا يوجد ربط دفع أو قاعدة بيانات.

## مصدر الصور
صور حقيقية من ملف النشاط على خرائط قوقل، مُنتقاة يدويًا. استبعدنا 3 صور (تمثال/لافتة منافس، واجهة نشاط مختلف مغلق، لافتات فواكه مجاورة) واعتمدنا 4 صور للمحل ومنتجاته. كل `<img>` يشير لملف موجود مع `alt` عربي وأبعاد و`lazy` لغير الهيرو.

## نتائج الاختبار
حزمة Playwright في `tests/site.spec.ts` (تشغيل: `npx playwright test`) تغطي: اتجاه RTL والعنوان، ظهور الهيرو والـ CTA، اقتباس تقييم 4.8/446، تحميل كل الصور مع alt، فتح/إغلاق قائمة الجوال ملء الشاشة، غياب التمرير الأفقي عند 390px، تحقّق النموذج وبناء رابط واتساب وحفظ localStorage، اللايت بوكس، وصحة JSON-LD (Florist + aggregateRating).
